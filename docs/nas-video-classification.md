# NAS 视频库分类方案

## 目标

本方案用于整理 NAS 中的大量视频文件，适用于以下内容：

- 学习视频
- 电影
- 电视剧
- 纪录片
- 综艺
- 动画、番剧
- 国产、港台、亚洲、欧美等不同地区来源的视频

核心原则是：**一级目录按内容类型划分，地区作为二级目录或辅助分类**。

这样做的好处是：

- 方便人工查找
- 方便 Plex、Emby、Jellyfin 等媒体服务器识别
- 避免电影、电视剧、纪录片混在同一个地区目录中
- 后续扩展简单，不容易越整理越乱

## 总体目录结构

建议在 NAS 上建立一个统一的视频根目录，例如：

```text
Video/
  电影/
  电视剧/
  纪录片/
  学习视频/
  综艺/
  动画/
  家庭与私人/
  临时下载/
  待整理/
```

如果 NAS 上已有类似 `Movies`、`TV Shows`、`Courses` 等目录，也可以保留英文名，但建议保持结构一致。

## 一级目录说明

### 电影

用于存放完整电影、网络电影、动画电影、演唱会电影等。

推荐结构：

```text
电影/
  华语/
    大陆/
    香港台湾/
    新马华语/
  亚洲/
    日本/
    韩国/
    印度/
    泰国/
    其他亚洲/
  欧美/
    美国/
    英国/
    法国/
    德国/
    欧洲其他/
    其他欧美/
  动画电影/
  演唱会与音乐电影/
  未整理/
```

电影建议每部单独一个文件夹：

```text
电影/
  欧美/
    美国/
      The Matrix (1999)/
        The Matrix (1999).mkv
        poster.jpg
        fanart.jpg
```

中文电影可以使用中文片名，也可以中英文并列：

```text
电影/
  华语/
    香港台湾/
      无间道 Infernal Affairs (2002)/
        无间道 Infernal Affairs (2002).mkv
```

### 电视剧

用于存放连续剧、迷你剧、网剧等。

推荐结构：

```text
电视剧/
  国产剧/
  港台剧/
  日剧/
  韩剧/
  欧美剧/
  泰剧/
  其他地区/
  未整理/
```

电视剧建议按剧名、季、集组织：

```text
电视剧/
  欧美剧/
    Breaking Bad (2008)/
      Season 01/
        Breaking Bad - S01E01.mkv
        Breaking Bad - S01E02.mkv
      Season 02/
        Breaking Bad - S02E01.mkv
```

国产剧如果没有季的概念，也仍然建议使用 `Season 01`，这样最兼容媒体服务器：

```text
电视剧/
  国产剧/
    漫长的季节 (2023)/
      Season 01/
        漫长的季节 - S01E01.mkv
        漫长的季节 - S01E02.mkv
```

### 纪录片

纪录片建议按主题分类，而不是只按地区分类。

推荐结构：

```text
纪录片/
  自然地理/
  历史人文/
  科技工业/
  社会纪实/
  传记人物/
  美食旅行/
  军事战争/
  BBC-NHK-国家地理/
  国产纪录片/
  未整理/
```

示例：

```text
纪录片/
  自然地理/
    Planet Earth II (2016)/
      Season 01/
        Planet Earth II - S01E01.mkv
```

国产纪录片：

```text
纪录片/
  国产纪录片/
    河西走廊 (2015)/
      Season 01/
        河西走廊 - S01E01.mkv
```

### 学习视频

学习视频通常不适合交给影视刮削器管理，建议单独建库，按主题和课程组织。

推荐结构：

```text
学习视频/
  编程开发/
    Python/
    Java/
    前端/
    数据库/
    运维与云计算/
  AI与数据/
    机器学习/
    深度学习/
    大模型/
    数据分析/
  设计剪辑/
    Photoshop/
    Premiere/
    After Effects/
    Blender/
  语言学习/
    英语/
    日语/
    韩语/
  财经商业/
    投资理财/
    管理课程/
    商业分析/
  考试认证/
  公开课/
  其他课程/
```

课程建议按课程名、章节、课时组织：

```text
学习视频/
  AI与数据/
    大模型/
      大模型应用开发实战/
        01-课程介绍/
          001-课程导学.mp4
          002-环境准备.mp4
        02-提示词工程/
          001-提示词基础.mp4
```

学习视频命名重点是清晰排序，建议统一使用三位数字：

```text
001-课程导学.mp4
002-环境准备.mp4
003-第一个项目.mp4
```

### 综艺

综艺建议独立出来，不要混在电视剧里。

推荐结构：

```text
综艺/
  大陆综艺/
  港台综艺/
  日韩综艺/
  欧美综艺/
  脱口秀/
  音乐舞台/
  未整理/
```

示例：

```text
综艺/
  大陆综艺/
    声生不息 (2022)/
      Season 01/
        声生不息 - S01E01.mkv
```

### 动画

动画是否单独建一级目录，取决于你的收藏规模。

如果动画很多，建议单独建：

```text
动画/
  日本番剧/
  国产动画/
  欧美动画/
  动画电影/
  儿童动画/
  未整理/
```

如果动画不多，也可以放入：

- 动画电影放入 `电影/动画电影/`
- 番剧放入 `电视剧/动画番剧/`

建议二选一，不要两边都放，避免重复和混乱。

### 家庭与私人

用于存放个人拍摄、家庭录像、生活记录等不适合刮削的内容。

推荐结构：

```text
家庭与私人/
  家庭录像/
  旅行记录/
  手机备份视频/
  重要留存/
```

这类内容建议单独设置权限，不要和公共影视库混在一起。

### 临时下载

用于下载软件、BT 工具、网盘同步工具的临时落点。

推荐结构：

```text
临时下载/
  正在下载/
  下载完成待检查/
  可删除/
```

建议不要让媒体服务器扫描 `临时下载`，否则会产生大量错误识别。

### 待整理

用于暂时无法判断类型、命名混乱、缺少信息的视频。

推荐结构：

```text
待整理/
  电影待确认/
  剧集待确认/
  课程待确认/
  重复待确认/
  低清待替换/
```

`待整理` 是缓冲区，不建议长期堆积。可以每周或每月集中处理一次。

## 推荐命名规范

### 电影命名

推荐格式：

```text
片名 (年份)/片名 (年份).扩展名
```

示例：

```text
Inception (2010)/Inception (2010).mkv
流浪地球 (2019)/流浪地球 (2019).mkv
无间道 Infernal Affairs (2002)/无间道 Infernal Affairs (2002).mkv
```

如果有版本信息，可以放在文件名后面：

```text
Inception (2010) - 2160p UHD BluRay HDR TrueHD Atmos.mkv
```

### 电视剧命名

推荐格式：

```text
剧名 (年份)/Season 01/剧名 - S01E01.扩展名
```

示例：

```text
Breaking Bad (2008)/Season 01/Breaking Bad - S01E01.mkv
漫长的季节 (2023)/Season 01/漫长的季节 - S01E01.mkv
```

双集文件可以这样命名：

```text
剧名 - S01E01-E02.mkv
```

特别篇可以这样命名：

```text
剧名 - S00E01 - 特别篇.mkv
```

### 纪录片命名

纪录片如果是多集，按电视剧方式命名：

```text
Planet Earth II (2016)/Season 01/Planet Earth II - S01E01.mkv
```

单集纪录片按电影方式命名：

```text
Free Solo (2018)/Free Solo (2018).mkv
```

### 学习视频命名

推荐格式：

```text
课程名/
  01-章节名/
    001-课时标题.mp4
    002-课时标题.mp4
```

示例：

```text
Python 零基础入门/
  01-环境准备/
    001-课程介绍.mp4
    002-安装 Python.mp4
  02-基础语法/
    001-变量与类型.mp4
```

## 地区分类建议

地区建议作为二级目录，而不是一级目录。

推荐地区划分：

```text
华语/
  大陆/
  香港台湾/
  新马华语/

亚洲/
  日本/
  韩国/
  印度/
  泰国/
  其他亚洲/

欧美/
  美国/
  英国/
  法国/
  德国/
  欧洲其他/
  其他欧美/
```

对于不好判断地区的内容，建议按以下优先级决定：

1. 按主要出品国家或地区
2. 按主要语言
3. 按你个人查找习惯
4. 实在不确定，放入 `未整理`

## 媒体服务器建议

如果使用 Plex、Emby 或 Jellyfin，建议按以下方式建立媒体库：

```text
媒体库：电影
路径：Video/电影
类型：电影

媒体库：电视剧
路径：Video/电视剧
类型：电视剧

媒体库：纪录片
路径：Video/纪录片
类型：电影或电视剧，视内容形态而定

媒体库：学习视频
路径：Video/学习视频
类型：家庭视频、其他视频或不刮削

媒体库：综艺
路径：Video/综艺
类型：电视剧或其他视频

媒体库：动画
路径：Video/动画
类型：电视剧和电影可分开建库
```

不要把 `临时下载`、`待整理` 加入媒体服务器扫描路径。

## 整理流程

推荐按以下顺序整理：

1. 先建立一级目录
2. 把明显类型的视频先移动到对应一级目录
3. 电影按地区分类
4. 电视剧按地区或语种分类
5. 学习视频按主题分类
6. 纪录片按主题分类
7. 对文件名进行统一命名
8. 检查重复文件和低清版本
9. 再接入媒体服务器扫描
10. 最后处理 `待整理`

## 重复文件处理建议

遇到重复资源时，建议保留优先级如下：

1. 分辨率更高
2. 码率更合理
3. 音轨更完整
4. 字幕更完整
5. 文件名更规范
6. 来源更可靠

不建议只按文件大小判断质量。部分体积大的文件可能是低效率编码，部分体积小的文件也可能是高质量 HEVC 编码。

## 清晰度和版本标记

常见标记建议保留在文件名中：

```text
1080p
2160p
BluRay
WEB-DL
REMUX
HDR
DV
Atmos
AAC
TrueHD
```

示例：

```text
Dune Part Two (2024) - 2160p UHD BluRay REMUX HDR TrueHD Atmos.mkv
```

如果只是普通观看，不必把所有参数都写得很长。目录名保持简洁，详细信息可以留在文件名里。

## 字幕管理

外挂字幕建议和视频同名：

```text
Movie Name (2020).mkv
Movie Name (2020).zh-CN.srt
Movie Name (2020).en.srt
```

电视剧字幕：

```text
剧名 - S01E01.mkv
剧名 - S01E01.zh-CN.srt
```

常用语言标记：

```text
zh-CN  简体中文
zh-TW  繁体中文
en     英文
ja     日文
ko     韩文
```

## 不建议的分类方式

不建议使用下面这种一级目录结构：

```text
国产/
港台/
亚洲/
欧美/
学习/
电影/
电视剧/
```

原因是地区和类型混在同一级，会产生冲突：

- 国产电影放 `国产` 还是 `电影`
- 欧美纪录片放 `欧美` 还是 `纪录片`
- 港台电视剧放 `港台` 还是 `电视剧`
- 日本动画电影放 `亚洲`、`动画` 还是 `电影`

长期来看，这种结构会越来越难维护。

## 执行建议

第一次整理不要追求一步到位。建议先完成三件事：

1. 建好一级目录
2. 把电影、电视剧、纪录片、学习视频分开
3. 把下载目录和待整理目录从媒体服务器扫描中排除

等主结构稳定后，再慢慢处理地区、年份、字幕、版本和重复文件。

最重要的是保持规则稳定。规则一旦确定，后续新增视频都按同一套方式进入目录，NAS 视频库才会越用越清楚。
