## Framework 设置持久化与 AI API Key 保存

- 用户要求：AI API Key 为什么没有保存下来？数据存在哪里？是不是 SQLite？
- 问题原因：安装版 `set_settings` 只写 Rust 内存，重启后丢失；浏览器预览 fallback 使用 `localStorage`，但桌面安装版没有持久化；当时框架模板尚未使用 SQLite 存设置。
- 关键操作：桌面安装版启动时读取 Tauri 应用数据目录下的 `settings.json`；每次 `set_settings` 写回 `settings.json`；`aiProfile.apiKey/baseURL/model/name` 与其它设置一起保存；SQLite 只用于操作日志。
- 测试结果：`pnpm build`、`cargo check --target x86_64-pc-windows-gnu --jobs 1`、`scripts/package-windows.ps1` 均通过。
- 数据位置：桌面设置文件为 `%APPDATA%/com.tauriframework.app/settings.json`；浏览器预览使用 `localStorage`。
- 结果：AI API Key 可在桌面安装版持久保存。
